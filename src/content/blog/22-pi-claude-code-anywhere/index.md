## 자취방 공유기 옆 라즈베리파이

자취방 공유기 옆에 라즈베리파이 한 대가 24시간 켜져 있다. 본래 목적은 [숙제 사이트](/projects/homework-site) 호스팅이었다. Django, SQLite, Nginx 정도의 워크로드는 파이 한 대로 충분하다.

근데 이 파이가 요즘 내 인생에서 가장 비중이 큰 컴퓨터가 됐다. 노트북도 데탑도 아니고, **이 손바닥만한 파이가**.

이유는 하나다. **여기에 Claude Code가 깔려 있고, 나는 어디서든 ssh로 들어갈 수 있다.**

---

## 텍스트 출력에 머무르는 AI의 한계

ChatGPT, Gemini, Claude 웹앱 — 전부 폰에서도 잘 돈다. 어디서든 쓸 수 있다. 근데 얘들이 할 수 있는 건 결국 **답을 글자로 뱉어주는 것까지**다.

- 내 컴퓨터의 파일을 직접 못 읽는다.
- 내 코드를 직접 못 고친다.
- 스크립트를 직접 못 돌린다.
- 백그라운드 작업을 직접 못 건다.

결국 사람이 중간에서 복붙 셔틀을 해야 한다. "이 코드 이렇게 고쳐줘" → 답 받음 → 내가 IDE 켜서 붙여넣음 → 저장 → 실행. AI가 똑똑한 만큼 내가 손이 바빠진다.

---

## Claude Code의 차이점, 권한

Claude Code는 다르다. 얘는 내 컴퓨터 안에서 돈다. 그리고 **권한을 다 가지고 있다.**

- 파일을 직접 읽고 쓴다.
- 셸을 직접 친다.
- `git commit`, `git push`를 직접 한다.
- 서버를 직접 띄우고 죽인다.
- cron, systemd 서비스를 직접 건다.
- 다른 컴퓨터에 ssh로 들어가서 거기서도 똑같이 한다.

ChatGPT가 **조언자**라면, Claude Code는 **실행자**에 가깝다.

---

## 파이 + Claude Code + ssh 의 조합

이 두 요소를 합치면 다음과 같은 구성이 가능하다.

1. 라즈베리파이는 24시간 켜져 있다.
2. 어디서든 ssh로 들어갈 수 있다 (외부망에서는 [WoL + DDNS](/blog/18-wol-setting), 폰에서는 [Termius](https://termius.com)).
3. 들어가서 `claude` 한 줄 치면 된다.

**컴퓨터 없이도 컴퓨터에서 Claude Code가 할 수 있는 모든 일을 할 수 있다.** 카페에서 폰만 들고 있어도 다음과 같은 일이 된다.

- 블로그 레포에 새 글 커밋하고 GH Pages 배포 트리거.
- homework_site 서버 로그 까보고 버그 픽스 PR까지.
- 새벽에 떠오른 사이드 프로젝트 아이디어를 그 자리에서 `npm create`부터 첫 커밋까지.
- 잘 모르는 분야의 논문 5개 받아서 비교 요약.
- 데탑을 [WoL로 깨워서](/blog/18-wol-setting) 데탑의 데이터까지 끌어다 처리.

노트북을 펼 필요가 없다. **내 폰 하나가 곧 작업 환경**이 된다.

## 맥미니 붐의 전제

오픈클로 같은 상시 가동형 에이전트 서비스가 화제일 때, 한 차례 **맥미니 붐**이 있었다. 월 구독 비용을 줄이는 대신 맥미니에 로컬 모델을 올려 같은 기능을 직접 구현하자는 흐름이었다.

맥미니가 후보로 거론된 이유는 디자인 요인이 아니라 **통합 메모리(Unified Memory)** 구조 때문이다.

### 큰 VRAM이 필요한 이유

LLM 을 돌리려면 **모델 가중치를 메모리에 통으로 올려놓아야 한다.** 토큰 하나 쓸 때마다 모델의 수백 억 개 파라미터를 전부 읽어야 하기 때문에, 디스크에서 조금씩 스와핑하면 초당 한 토큰도 안 나온다. **엄청 빠른 메모리(= VRAM)에 상주해야** 쓸만한 속도가 나온다.

대략적인 메모리 요구는 다음과 같다.

- 7B 모델 (양자화) — 약 5GB VRAM
- 32B 모델 (양자화) — 약 20GB VRAM
- 70B 모델 (양자화) — 약 40GB VRAM
- Claude Sonnet급 — 더 올라간다 (가중치 자체가 비공개)

근데 소비자용 그래픽 카드는 VRAM이 작다. RTX 4090도 24GB, RTX 5090이 32GB. **70B 모델 올리려면 그래픽 카드 두 장 꽂아서 수백만 원이 사라진다.**

여기서 맥의 통합 메모리가 빛을 본다. 맥은 RAM과 VRAM이 분리되어 있지 않고, CPU와 GPU가 메모리를 통으로 공유한다. **맥미니 64GB 모델을 사면 그 64GB 거의 전부를 모델 로딩에 퍼부을 수 있다.** RTX 5090 한 장이 못 올리는 70B 모델이 맥미니에서는 올라간다. 그것도 그래픽 카드 두 장 박는 것보다 훨씬 싼 가격으로. "로컬 LLM 용으로는 맥이 가성비 좋다"는 공식이 자리잡은 배경이다.

### 실제로 돌려보면

[Ollama](https://ollama.com)로 로컬 모델들을 직접 돌려봤다. Llama 3, Qwen, DeepSeek 계열의 양자화된 7B부터 32B까지.

결론은 단순했다. **소비자용 하드웨어에서 돌릴 수 있는 로컬 모델은 아직 Claude Code의 자리를 대체할 수준이 전혀 아니다.** 간단한 요약, 분류, 난이도 낮은 Q&A 정도는 괜찮은데, "이 레포 돌아다니면서 버그 찾고 PR 올려" 같은 **에이전트성 작업은 도저히 못 따라온다.** 프런티어 모델과 격차가 너무 크다. RTX 5090급 하드웨어에서도 비슷한 결론이었다.

맥미니는 하드웨어 자체로는 더 우수하다. 64GB 모델이라면 70B도 올라간다. 하지만 **그 위에 올릴 모델이 애초에 쓸 수준이 아니니** 결론은 똑같다. 통합 메모리 프리미엄으로 100만 원을 쓰고도 결국 **Claude Sonnet/Opus는 못 돌리는 기계**를 손에 쥐게 되는 거다.

프런티어 모델이 필요한 작업은 결국 클라우드 API를 부르게 된다. 그리고 클라우드 API를 부를 거면 애초에 밑에 깔린 메모리 괴물이 필요가 없다.

## 제안 구성, 라즈베리파이 + Claude Max

같은 목적, 즉 어디서든 돌아가는 개인 에이전트 환경을 더 가성비 좋게 구성하는 방법은 이거다.

- 라즈베리파이 5 (8GB) — **약 15만 원**
- Claude Max 구독 — 어차피 코딩용으로 쓰고 있는 자원
- (선택) 텔레그램 봇 — 추가 비용 없음, [이전 포스트](/projects/telegram-claude-bot) 참고

파이는 로컬 모델을 돌리는 게 아니다. 파이는 **세션을 제공한다.** Claude Code는 어차피 클라우드에 있는 Sonnet/Opus를 API로 부르게 되어 있으니, **에이전트가 "살고 있을" 서버만 있으면 된다.** 그 서버가 70B 모델을 먹이는 괴물일 필요는 전혀 없다.

15만 원 파이 한 대 + 이미 내고 있는 Claude Max 구독 = **100만 원짜리 맥미니가 제대로 못 하는 일을 더 잘한다.**

---

## 한 가지 단점, 모바일 ssh UI

이 구성에는 한 가지 단점이 있다. 휴대폰에서 Termius 같은 ssh 앱으로 Claude Code를 직접 사용하면 UX가 떨어진다. 좁은 화면에서 터미널 글자가 작고, 한국어 IME와 터미널 키바인딩 충돌이 발생하며, 네트워크가 끊기면 세션이 종료된다.

이 부분은 [텔레그램 봇](/projects/telegram-claude-bot)으로 일정 부분 우회할 수 있다. 폰에서 텔레그램으로 자연어 한 줄을 보내면, 파이의 봇이 그것을 `claude -p`로 넘겨 실행하고 결과를 다시 텔레그램으로 답장한다. ssh 앱을 켤 필요도 없다.

작업 성격에 따라 두 경로를 분리해서 쓴다.

- **가벼운 명령** — 텔레그램 한 줄.
- **세션이 필요한 깊은 작업** — Termius로 ssh 들어가서 `claude`.

둘 다 라즈베리파이 한 대 위에서 돌고, 둘 다 폰에서 된다.

## 정리

- 맥미니 붐은 **통합 메모리로 대용량 모델을 올릴 수 있다**는 전제 위에 서 있었다.
- 근데 그 위에서 돌릴 수 있는 로컬 모델은 아직 Claude Sonnet/Opus와 격차가 너무 크다. RTX 5090급이어도 마찬가지.
- 그러면 통합 메모리의 이점은 사실상 살릴 데가 없는 소비가 된다.
- 그럴 바엔 **15만 원 라즈베리파이 한 대 + 이미 쓰는 Claude Max 구독**으로 그 자리를 메꾸는 게 가성비가 훨씬 좋다. 어차피 클라우드의 프런티어 모델을 부를 거면 서버는 그냥 "살아있기만" 하면 된다.
- 어디서든 ssh / 텔레그램으로 트리거.
- ChatGPT가 못 하는 "실제 컴퓨터를 만지는 일"을 폰만 들고도 시킨다.

2년 전이었으면 "개인 비서 AI" 같은 거 SF 영역이었다. 지금은 **달랑 파이 하나랑 구독 하나**면 된다. 내가 자랑하고 싶은 게 좋은 장비가 아니라 **이게 가능해진 시대**라는 점에서 좀 신기한 글이다.
