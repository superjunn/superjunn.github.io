## 집 공유기 옆 라즈베리파이

우리 집 공유기 옆에는 [라즈베리파이](/projects/home-lab-setup) 한 대가 24시간 켜져 있다. 원래 [숙제 사이트](/projects/homework-site) 호스팅하려고 둔 거다. Django + SQLite + Nginx 정도는 파이로 넘치고도 남는다 — 굳이 전기 많이 먹는 서버컴 살 이유가 없었다.

근데 이놈이 요즘 내 인생에서 가장 비중이 큰 컴퓨터가 됐다. 노트북도 데탑도 아니고 **이 손바닥만한 파이가**.

이유는 하나다. **여기에 Claude Code가 깔려 있고, 나는 어디서든 ssh로 들어갈 수 있다.**

---

## "텍스트만 뱉는 AI" 의 한계

ChatGPT, Gemini, Claude 웹앱 — 전부 폰에서도 잘 돈다. 어디서든 쓸 수 있다. 근데 얘들이 할 수 있는 건 결국 **답을 글자로 뱉어주는 것까지**다.

- 내 컴퓨터의 파일을 직접 못 읽는다.
- 내 코드를 직접 못 고친다.
- 스크립트를 직접 못 돌린다.
- 백그라운드 작업을 직접 못 건다.

결국 사람이 중간에서 복붙 셔틀을 해야 한다. "이 코드 이렇게 고쳐줘" → 답 받음 → 내가 IDE 켜서 붙여넣음 → 저장 → 실행. AI가 똑똑한 만큼 내가 손이 바빠진다.

## Claude Code 가 다른 이유 — **권한**

Claude Code는 다르다. 얘는 내 컴퓨터 안에서 돈다. 그리고 **권한을 다 가지고 있다.**

- 파일을 직접 읽고 수정한다.
- `bash` 를 직접 친다.
- `git commit`, `git push` 를 직접 한다.
- 서버를 직접 띄우고 죽인다.
- cron job, systemd 서비스를 직접 건다.
- 다른 컴퓨터에 ssh로 들어가서 거기서도 똑같이 한다.

이게 ChatGPT랑 Claude Code의 차이다. 전자는 **조언자**, 후자는 **실행자**.

## 그래서 — ssh 로 파이에 박힌 Claude Code 를 돌리면

자, 이 두 개를 합치면 어떻게 될까.

1. 라즈베리파이는 24시간 켜져 있다.
2. 어디서든 ssh로 들어갈 수 있다 (집 밖이면 [WoL + DDNS](/blog/18-wol-setting), 폰이면 [Termius](https://termius.com)).
3. 들어가서 `claude` 한 줄 치면 된다.

**컴퓨터 없이도 컴퓨터에서 Claude Code가 할 수 있는 모든 일을 할 수 있다.** 카페에서 폰만 들고 있어도:

- 내 블로그 레포에 새 글 커밋하고 GH Pages 배포 트리거.
- homework_site 서버 로그 까보고 버그 픽스 PR 까지.
- 새벽에 떠오른 사이드 프로젝트 아이디어를 그 자리에서 `npm create` 부터 첫 커밋까지.
- 전혀 모르는 분야의 논문 5개 다운받아서 비교 요약.
- 데탑을 [WoL로 깨워서](/blog/18-wol-setting) 거기 있는 데이터까지 끌어와 처리.

노트북을 열 필요가 없다. **내 주머니가 곧 작업 환경**이 된다.

## 잠깐 — "오픈클로 돌리려고 맥미니" 붐의 정체

[오픈클로](https://openclod.com) 같은 "24시간 일하는 AI 비서" 서비스가 화제일 때 한 차례 **맥미니 붐**이 일었다. "오픈클로 같은 게 좋긴 한데 월 구독료가 너무 부담이니까, 맥미니 한 대 사서 로컬로 돌리면 어떨까?" 하는 아이디어.

근데 왜 하필 맥미니였을까. 이건 맥이 이뻐서가 아니라 **통합 메모리 (Unified Memory)** 때문이다.

### 왜 큰 VRAM 이 필요한가

LLM 을 돌리려면 **모델 가중치를 메모리에 통으로 올려놓아야 한다.** 토큰 하나 쓸 때마다 모델의 수백 억개 파라미터를 전부 읽어야 하기 때문에, 디스크에서 조금씩 스와핑하면 초당 한 토큰도 안 나온다. **엄청 빠른 메모리 (= VRAM) 에 상주해야** 쓸만한 속도가 나온다.

그래서 대충 이런 공식:

- 7B 모델 (양자화) → 약 5GB VRAM
- 32B 모델 (양자화) → 약 20GB VRAM
- 70B 모델 (양자화) → 약 40GB VRAM
- Claude Sonnet 급으로 가려면 → 더더 올라간다 (그리고 애초에 공개도 안 되어 있고)

근데 소비자용 그래픽 카드는 VRAM 이 작다. RTX 4090 도 24GB, RTX 5090 가 32GB. **70B 모델 올리려면 그래픽 카드 두 장 꽂아서 수백만원이 사라진다.**

여기서 맥의 통합 메모리가 빛을 본다. 맥은 RAM 과 VRAM 이 분리되어 있지 않고, CPU/GPU 가 메모리를 통으로 공유한다. **맥미니 64GB 모델을 사면 그 64GB 거의 전부를 모델 로딩에 퍼부을 수 있다.** RTX 5090 한 장이 못 올리는 70B 모델이 맥미니에서는 올라간다. 그것도 그래픽 카드 두 장 박는 것보다 훨씬 싼 가격으로. 그래서 "로컬 LLM 용으로는 맥이 가성비 좋다" 는 공식이 생겼고, "맥미니로 로컬 오픈클로" 가 말이 되는 것처럼 들렸던 거다.

### 근데 실제로 돌려보면

나도 궁금해서 [Ollama](https://ollama.com) 로 로컬 모델들을 직접 돌려봤다. Llama 3, Qwen, DeepSeek… 양자화된 7B부터 좋다는 32B까지.

결론은 명확했다 — **소비자용 하드웨어에서 돌릴 수 있는 로컬 모델은 아직 Claude Code 의 자리를 대체할 수준이 전혀 아니다.** 간단한 요약, 분류, 난이도 낮은 Q&A 정도는 괜찮은데, "이 레포 돌아다니면서 버그 찾고 PR 올려" 같은 **에이전트용 작업은 도저히 못 따라온다.** Frontier 모델과 격차가 너무 크다. RTX 5090 급 하드웨어도 돈 쓰고 쓰는 게 아깝다 싶을 정도.

맥미니는… 그래, 하드웨어는 더 좋다. 70B도 올라간다. 하지만 **그 위에 올릴 모델이 애초에 쓸 수준이 아니니** 결론은 똑같다. "통합 메모리 프리미엄"으로 100만원을 쓰고도 결국 **Claude Sonnet/Opus 는 못 돌리는 기계**를 손에 쥐고 있게 되는 거다.

그래서 맥미니로 오픈클로 돌리겠다 하는 건 내 입장에서는 돈지랄이다. **프론티어 모델이 필요한 일에는 어차피 클라우드 API 를 부르게 되고, 클라우드 API 를 부를 거면 애초에 밑에 깔린 메모리 괴물이 필요가 없으니까.**

## 제안 — 라즈베리파이 + Claude Max

동일한 목적 (= 언제 어디서든 돌아가는 나만의 에이전트) 을 달성하는 더 가성비 좋은 방법은 이거다:

- 라즈베리파이 5 (8GB) — **약 15만원**
- Claude Max 구독 — 어차피 따로 코딩용으로 쓰고 있음
- (선택) 텔레그램 봇 — 공짜, [이전 포스트](/projects/telegram-claude-bot) 참고

파이는 로컬 모델을 돌리는 게 아니다. 파이는 **세션을 제공한다.** Claude Code 는 어차피 클라우드에 있는 Sonnet/Opus 를 API로 부르게 되어 있으니, **에이전트가 "살고 있을" 서버만 있으면 된다.** 그 서버가 70B 모델을 먹이는 괴물일 필요가 전혀 없다.

15만원 파이 한 대 + 이미 내고 있는 Claude Max 구독 = **100만원짜리 맥미니가 제대로 못 하는 일을 더 잘한다.**

## 단 하나 단점 — 모바일 ssh UI

솔직히 단점도 있다. 폰에서 Termius 같은 ssh 앱으로 직접 Claude Code 를 돌리면 UI가 좀 별로다. 좁은 화면에서 터미널 글자 보기 빡세고, 한글 IME랑 터미널 키바인딩이랑 부딪히고, 네트워크 끊기면 세션 날아가고.

근데 이것도 [예전에 글 썼듯이](/projects/telegram-claude-bot) **텔레그램 봇으로 우회**할 수 있다. 폰에서 텔레그램으로 자연어 한 줄 보내면 파이의 봇이 그걸 `claude -p` 로 넘겨서 실행하고 결과를 다시 텔레그램으로 답장한다. ssh 앱 켤 필요도 없다.

결국 이 조합이면:

- **가벼운 명령** — 텔레그램 한 줄.
- **세션이 필요한 깊은 작업** — Termius로 ssh 들어가서 `claude`.

둘 다 라즈베리파이 한 대 위에서 돌고, 둘 다 폰에서 된다.

## 정리

- 맥미니 붐은 **통합 메모리로 대용량 모델을 올릴 수 있다**는 전제 위에 서 있었다.
- 근데 그 위에서 돌릴 수 있는 로컬 모델은 아직 Claude Sonnet/Opus 와 격차가 너무 크다 — RTX 5090 급이어도 마찬가지.
- 그러면 통합 메모리의 이점은 사실상 살릴 데가 없는 소비가 된다.
- 그럴 바엔 **15만원 라파 한 대 + 이미 쓰는 Claude Max 구독** 으로 그 자리를 메꾸는 게 가성비가 훨씬 좋다. 어차피 클라우드의 프론티어 모델을 부를 거면 서버는 그냥 "살아있기만" 하면 된다.
- 어디서든 ssh / 텔레그램으로 트리거.
- ChatGPT가 못 하는 "실제 컴퓨터를 만지는 일"을 폰만 들고도 시킨다.

2년 전이었으면 "개인 비서 AI" 같은 거 SF 영역이었다. 지금은 **달랑 파이 하나랑 구독 하나**면 된다. 내가 자랑하고 싶은 게 좋은 장비가 아니라 **이게 가능해진 시대**라는 점에서 좀 신기한 글이다.
