> 이 글은 [15만원 라즈베리파이로 100만원 맥미니 대체하기](/blog/22-pi-claude-code-anywhere) 와 동일한 내용을 다른 톤으로 다시 써본 비교용 글이다. 어느 톤이 더 읽기 좋은지 판단하기 위해 잠시 두 글을 함께 두었다.

## 자취방 공유기 옆 라즈베리파이

자취방 공유기 옆에 라즈베리파이 한 대가 24시간 켜져 있다. 본래 목적은 [숙제 사이트](/projects/homework-site) 호스팅이었다. Django, SQLite, Nginx 정도의 워크로드는 파이 한 대로 충분하다.

이 파이가 최근 들어 내 작업 환경에서 가장 자주 접속하는 컴퓨터가 되었다. 노트북도 데탑도 아니다. 이유는 한 가지로 정리된다. 파이에 Claude Code가 설치되어 있고, 어디서든 ssh로 접속할 수 있기 때문이다.

---

## 텍스트 출력에 머무르는 AI의 한계

ChatGPT, Gemini, Claude 웹앱은 모두 모바일에서도 동작하고, 휴대성이 좋다. 다만 이들이 수행할 수 있는 작업의 범위는 텍스트 응답을 제공하는 수준에 머무른다.

- 사용자의 컴퓨터 파일을 직접 읽거나 수정할 수 없다.
- 코드를 직접 변경할 수 없다.
- 스크립트를 직접 실행할 수 없다.
- 백그라운드 작업을 직접 트리거할 수 없다.

결과적으로 사용자가 중간에서 복사·붙여넣기를 반복하게 된다. 응답을 받아 IDE에 붙이고, 저장하고, 실행하는 흐름이다. 모델의 정확도와 무관하게 사용자의 손이 늘어나는 구조다.

---

## Claude Code의 차이점, 권한

Claude Code는 사용자의 컴퓨터 위에서 직접 동작하며, 파일·셸·네트워크 권한을 모두 갖는다.

- 파일을 직접 읽고 쓴다.
- 셸 명령을 직접 실행한다.
- git commit, git push를 직접 수행한다.
- 서버 프로세스를 직접 띄우고 종료한다.
- cron, systemd 같은 시스템 자원을 직접 관리한다.
- 다른 머신에 ssh로 접속해 동일한 작업을 수행한다.

ChatGPT가 조언자에 가깝다면, Claude Code는 실행자에 가깝다고 본다.

---

## 파이 + Claude Code + ssh 의 조합

이 두 요소가 결합되면 다음과 같은 구성이 가능하다.

1. 라즈베리파이는 24시간 가동 상태다.
2. 어디서든 ssh로 접속할 수 있다 (외부망에서는 [WoL + DDNS](/blog/18-wol-setting), 모바일에서는 [Termius](https://termius.com)).
3. 접속 후 `claude` 한 줄로 작업을 시작한다.

작업 가능한 범위는 데스크탑에서 Claude Code로 할 수 있는 작업과 거의 동일하다. 카페에서 휴대폰만 가지고 있는 상황을 가정하면 다음과 같은 일이 가능하다.

- 블로그 레포에 새 글을 커밋하고 GH Pages 배포를 트리거하기
- homework_site 서버 로그를 확인하고 버그 수정 PR을 작성하기
- 떠오른 사이드 프로젝트 아이디어를 `npm create`부터 첫 커밋까지 진행하기
- 잘 모르는 분야의 논문 다섯 편을 받아 비교 요약하기
- 데탑을 [WoL로 깨워서](/blog/18-wol-setting) 데탑의 데이터까지 끌어다 처리하기

노트북을 펴지 않은 상태에서 휴대폰만으로 동일한 작업이 가능하다는 의미다.

## 맥미니 붐의 전제

오픈클로 같은 상시 가동형 에이전트 서비스가 화제일 때, 한 번의 맥미니 붐이 있었다. 월 구독 비용을 줄이는 대신 맥미니에 로컬 모델을 올려 같은 기능을 직접 구현하자는 흐름이었다.

맥미니가 후보로 거론된 이유는 디자인 요인이 아니라 통합 메모리(Unified Memory) 구조 때문이다.

### 큰 VRAM이 필요한 이유

LLM은 토큰을 생성할 때마다 모델 가중치 전체에 접근해야 한다. 디스크에서 부분적으로 스와핑하는 방식으로는 초당 한 토큰도 생성하기 어렵다. 가중치는 빠른 메모리(VRAM) 위에 상주해야 한다.

대략적인 메모리 요구 사항은 다음과 같다.

- 7B 모델 (양자화) — 약 5GB VRAM
- 32B 모델 (양자화) — 약 20GB VRAM
- 70B 모델 (양자화) — 약 40GB VRAM
- Claude Sonnet급 — 더 큰 메모리가 필요하며, 가중치 자체가 비공개다

소비자용 GPU의 VRAM은 이에 비해 작다. RTX 4090이 24GB, RTX 5090이 32GB다. 70B 모델을 올리려면 GPU를 두 장 구성해야 하며 비용이 수백만 원 단위로 늘어난다.

이 지점에서 맥의 통합 메모리가 의미를 갖는다. 맥은 RAM과 VRAM을 분리하지 않고 CPU와 GPU가 공유한다. 64GB 모델 맥미니라면 그 64GB 거의 전부를 모델 로딩에 할당할 수 있다. RTX 5090 한 장으로 올리지 못하는 70B 모델이 맥미니에서는 동작하며, GPU 두 장을 구성하는 것보다 비용이 낮다. 로컬 LLM 용도에서 맥의 가성비가 좋다는 평가가 자리잡은 배경이다.

### 실제 사용 결과

[Ollama](https://ollama.com)로 다양한 로컬 모델을 직접 실행해 보았다. Llama 3, Qwen, DeepSeek 계열의 7B에서 32B까지 양자화 버전을 시도했다.

결론은 단순했다. 소비자용 하드웨어에서 동작하는 로컬 모델은 현시점에서 Claude Code의 역할을 대체할 수준에 도달하지 못한 것으로 보인다. 간단한 요약, 분류, 단순 Q&A 정도는 가능하지만, 레포지토리를 순회하며 버그를 찾아 PR을 작성하는 식의 에이전트성 작업에서는 격차가 크게 드러난다. RTX 5090급 하드웨어에서도 비슷한 결론이 나왔다.

맥미니 64GB의 경우 70B 모델까지 올릴 수 있다는 점에서 하드웨어 자체는 더 우수하다. 다만 그 위에 올릴 수 있는 모델이 아직 실용 수준에 도달하지 못했다는 점에서 결론은 같다. 통합 메모리 프리미엄으로 100만 원을 지출하고도, Claude Sonnet/Opus급 모델은 구동할 수 없는 상태가 된다.

프런티어 모델이 필요한 작업은 결국 클라우드 API를 호출하게 되며, 클라우드 API를 호출하는 시점에서 로컬의 메모리 용량은 의미를 잃는다.

## 제안 구성, 라즈베리파이 + Claude Max

동일한 목적, 즉 어디서나 동작하는 개인 에이전트 환경을 더 낮은 비용으로 구성하는 방법은 다음과 같다.

- 라즈베리파이 5 (8GB) — 약 15만 원
- Claude Max 구독 — 별도 코딩 용도로 이미 사용 중인 자원
- (선택) 텔레그램 봇 — 추가 비용 없음, [이전 포스트](/projects/telegram-claude-bot) 참고

파이의 역할은 로컬 모델 실행이 아니라 세션 호스팅이다. Claude Code는 클라우드의 Sonnet/Opus를 API로 호출하므로, 에이전트가 거주할 항상 켜져 있는 서버만 있으면 충분하다. 그 서버가 대용량 모델을 적재할 수 있는 사양일 필요는 없다.

15만 원 파이 한 대와 이미 사용 중인 Claude Max 구독을 결합하면, 100만 원대 맥미니가 수행하지 못하는 작업을 더 안정적으로 처리할 수 있다고 본다.

---

## 한 가지 단점, 모바일 ssh UI

이 구성에는 한 가지 단점이 있다. 휴대폰에서 Termius 같은 ssh 앱으로 Claude Code를 직접 사용하면 UX가 떨어진다. 좁은 화면에서 터미널 글자가 작고, 한국어 IME와 터미널 키바인딩 충돌이 발생하며, 네트워크가 끊기면 세션이 종료된다.

이 부분은 [텔레그램 봇](/projects/telegram-claude-bot)으로 일정 부분 우회할 수 있다. 휴대폰에서 텔레그램으로 자연어 메시지를 전송하면 파이의 봇이 이를 `claude -p`로 전달하고, 결과를 다시 텔레그램으로 응답한다. ssh 앱을 직접 띄우지 않아도 된다.

작업 성격에 따라 두 경로를 분리해서 사용한다.

- 짧은 명령은 텔레그램으로 전송한다.
- 세션이 필요한 작업은 Termius로 ssh 접속 후 `claude`를 실행한다.

두 경로 모두 라즈베리파이 한 대 위에서 동작하며, 휴대폰만으로 사용 가능하다.

## 정리

- 맥미니 붐은 통합 메모리로 대용량 모델을 적재할 수 있다는 전제에 의존했다.
- 그 전제 위에서 실제로 사용 가능한 로컬 모델은 현재 Claude Sonnet/Opus급과 격차가 크다. RTX 5090급 하드웨어에서도 동일한 결론에 도달한다.
- 그 결과 통합 메모리의 이점이 실용적으로 활용될 영역이 줄어든다.
- 대안으로 라즈베리파이와 Claude Max 구독의 조합이 더 합리적이라고 본다. 클라우드 프런티어 모델을 호출하는 구조라면 서버는 항상 켜져 있는 정도면 충분하다.
- 트리거 경로는 ssh 또는 텔레그램으로 통일된다.
- ChatGPT가 처리하지 못하는 실제 시스템 조작 작업을 휴대폰만으로 수행할 수 있다.

2년 전 시점에서는 개인용 AI 비서가 SF의 영역에 가까웠다. 현재는 라즈베리파이 한 대와 구독 하나로 동일한 사용 경험이 가능해졌다. 인상 깊은 부분은 장비의 사양이 아니라, 그 정도 자원으로 이런 구성이 가능한 시대에 도달했다는 점이라고 생각한다.
